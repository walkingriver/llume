The Problem

Ask an LLM to build the same feature twice in React or Angular, and you'll get different implementations each time. Flexibility creates ambiguity. Ambiguity creates hallucination.

The Solution

LLasM is an agent skill‚Äîa set of instructions that AI coding agents read and follow. One way to handle events. One way to bind state. One manifest format. No choices means no hallucination.

A Skill File

SKILL.md contains the rules. Install it, and your AI agent knows exactly how to generate valid output.

A Tiny Runtime

llasm.js (~9KB gzipped) handles reactivity, routing, i18n, and progressive enhancements. Not a framework‚Äîjust glue.

Zero Build

No npm. No webpack. No Vite. The "build step" is the LLM generating HTML. Open it in a browser. Done.

Why Zero Build?

Every modern web framework assumes a build step: npm install, configure bundler, transpile TypeScript, tree-shake dependencies, optimize for production. That toolchain exists because humans need abstractions.

But when an LLM generates code, it can output the final format directly. No transpilation needed‚Äîit writes ES modules. No bundling needed‚Äîit inlines what's required. No tree-shaking needed‚Äîit only generates what's used.

The build step was always a human accommodation. Remove the human, remove the build.

LLasM will never require npm, webpack, Vite, Babel, or any CLI. It's pure ESM that runs directly in browsers. The "build step" is the LLM generating the HTML.

How It Works

1
Install the Skill

Run npx skills add walkingriver/llasm to teach your AI agent the LLasM instruction set. (Coming soon)

2
Ask for What You Want

"Build me a todo app with dark mode and persistence." The agent reads the skill and generates valid output.

3
Open in Browser

You get a complete HTML file + llasm.js. No npm install. No build step. Just open it.

The Output Format

Every LLasM page has exactly three parts

<!DOCTYPE html>
<html lang="en">
<body class="p3">
  <!-- 1. STATIC HTML -->
  <h1 data-m-tx="title" class="t6 tb c1"></h1>
  <button data-m-on="click:save">Save</button>
  
  <!-- 2. MANIFEST -->
  <script type="application/llasm+json">
    {"v":1,"r":{"s":{"count":0}},"l":{"en":{"title":"Hello"}}}
  </script>
  
  <!-- 3. HANDLERS -->
  <script type="module">
    import{l}from"./llasm.js";
    l.h({save:(e,s,L)=>L.t('Saved!','ok')});
  </script>
</body>
</html>
