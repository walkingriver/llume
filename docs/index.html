The Web Framework for
AI Coding Agents

LLasM is the first framework whose only intended user is an LLM. Humans don't read it. Humans don't write it. Humans don't debug it.

npx skills add walkingriver/llasm

Works with Cursor, Claude Code, Copilot, Windsurf, and more

Zero Build

No npm. No webpack. No Vite. Pure ESM that runs directly in browsers. The "build step" is the LLM generating HTML.

One Way

No flexibility. No choices. No escape hatches. There is exactly one way to do most things. Rigidity eliminates hallucination.

<25KB Runtime

Reactivity, routing, i18n, theming, persistence, toasts, modals, tabs, accordions‚Äîall in a runtime smaller than most hero images.

How It Works

1
Install the Skill

Run npx skills add walkingriver/llasm to teach your AI agent the LLasM instruction set.

2
Ask for What You Want

"Build me a todo app with dark mode and persistence." The agent reads the skill and generates valid output.

3
Open in Browser

You get a complete HTML file + llasm.js. No npm install. No build step. Just open it.

The Output Format

Every LLasM page has exactly three parts

<!DOCTYPE html>
<html lang="en">
<body class="p3">
  <!-- 1. STATIC HTML -->
  <h1 data-m-tx="title" class="t6 tb c1"></h1>
  <button data-m-on="click:save">Save</button>
  
  <!-- 2. MANIFEST -->
  <script type="application/llasm+json">
    {"v":1,"r":{"s":{"count":0}},"l":{"en":{"title":"Hello"}}}
  </script>
  
  <!-- 3. HANDLERS -->
  <script type="module">
    import{l}from"./llasm.js";
    l.h({save:(e,s,L)=>L.t('Saved!','ok')});
  </script>
</body>
</html>

Server-Side Skills

LLasM also supports full-stack development with native Node.js

REST API

Generate zero-dependency REST APIs using native Node.js http module. No Express, no Fastify‚Äîjust single-file servers.

const routes = {
  'GET /api/items': handler,
  'POST /api/items': handler,
  'DELETE /api/items/:id': handler
};

Run with: node server.js

SSR

Server-side render LLasM pages with dynamic state injection. File-based routing where each HTML file is a route.

const ssrHandlers = {
  '/': async () => ({ user, items }),
  '/products.html': async () => fetchProducts()
};

Run with: node ssr-server.js
